今天上午我们讲了activiMQ，
activeMQ 是一种开源的，实现了 JMS 规范的，面向消息的中间件，
为应用程序提供高效的、可扩展的、稳定的和安全的企业级消息通信。
主要是利用多线程来实现线程通信，及消息队列功能。

消息中间件利用高效可靠的消息传递机制进行平台无关的数据交流，
并基于数据通信来进行分布式系统的集成。
通过提供消息传递和消息排队模型，它可以在分布式环境下扩展进程间的通信。
对于消息中间件来说，常见的角色大致有生产者和消费者。

JMS是Java平台上有关面向消息中间件的技术规范，
它便于消息系统中的Java应用程序进行消息交换,
并且通过提供标准的产生、发送、接收消息的接口简化企业应用的开发。
JMS本身定义了一系列的接口规范，是一种与厂商无关的 API，用来访问消息收发系统。

activiMQ可以发送五种消息类型：
TextMessage  一个字符串对象 string
MapMessage  一套名称-值对
ObjectMessage  一个序列化的 Java 对象
BytesMessage  一个字节的数据流
StreamMessage  Java 原始值的数据流
mq中的角色有两种：分别是消息生产者和消息消费者
MQ中发送消息的方式有两种，点对点  和发布/订阅
点对点：一个消息生产者只能对应一个消费者，
如果消息生产者发送完消息后，没有消费者消费，消息会保存到队列中
发布订阅：一个或多个生产者对应一个或多个消费者，
如果消息生产者产生消息并发送完毕后，没有消费者消费，
那么消息会丢失
activiMQ的端口号有两个 分别是8161和61616

activiMQ的使用场景主要用于降低我们项目的耦合度。
